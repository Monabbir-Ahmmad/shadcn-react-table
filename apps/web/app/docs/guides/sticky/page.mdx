# Sticky header & footer

The header row (`enableStickyHeader`, default `true`) sticks to the top of the scroll surface, and the aggregation/footer row (`enableStickyFooter`, default `true`) sticks to the bottom. As you scroll through rows, both stay pinned in view.

## Bounded surface height

Sticky behavior requires a **bounded** scroll surface — there is nothing to stick to if the table grows to fit all its rows. While `enableStickyHeader` is on (the default), the surface is automatically capped near the viewport height (`clamp(350px, 100vh - 200px, 9999px)`), so tall tables scroll internally out of the box. To pick your own bound, set a max height with `surfaceClassName` on `<DataTable />` — it overrides the default:

```tsx
const table = useDataTable({
  data,
  columns,
  getRowId: (row) => row.id,
  enableStickyHeader: true,
  enableStickyFooter: true,
})

return <DataTable table={table} surfaceClassName="max-h-[460px]" />
```

Once the content overflows the bound, the header pins to the top and the footer pins to the bottom while the rows scroll between them. Setting `enableStickyHeader: false` removes both the pinning and the default bound, letting the table grow to its full height.

## Footer cells

Footer cells are populated from each column's `footer` definition or from aggregation values. See [Grouping](/docs/guides/grouping) for how aggregation rows are produced.

<Callout variant="note" title="Nothing to stick to?">
If the header or footer is not sticking, the surface is probably not bounded — check that `surfaceClassName` isn't removing the default max-height (e.g. `max-h-none`) and that the data actually overflows the bound.
</Callout>
